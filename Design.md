# 🎨 Glocalizer Design.md

> Plan.md(공모전 계획서) 기반 디자인 명세서.
> 여기 적힌 값들은 "출발점"이며, 실제 화면을 보면서 팀 피드백으로 다듬는다.
> ✏️ 표시 = 팀 취향에 따라 바꿔도 되는 항목

## 1. 디자인 방향 (컨셉)

**한 줄 정의: "완전히 토스(toss.im) 느낌 — 초록 포인트 하나, 큰 여백, 큰 글씨, 친근한 말투"**

- **토스가 유일한 기준**: 금융 서비스를 저렇게 쉬워 보이게 만든 그 문법을 그대로 — 흰 바탕, 연회색 면으로 구획, 둥근 카드, 큼직한 타이틀
- **"AI 서비스티" 절대 금지**: 보라색 그라데이션, ✨ 이모지, "마법처럼", 네온/우주 배경 같은 전형적 AI 클리셰 전부 배제. AI는 기능으로 증명하고 겉모습은 담담하게
- 심사위원이 처음 봐도 3초 안에 "아, 이모티콘을 번역해주는 서비스구나"를 알 수 있게

### 참고 레퍼런스
- **전부**: 토스(toss.im) — 레이아웃, 색, 말투, 버튼, 여백 전부 이 기준
- **업로드 UX만**: remove.bg — 드래그 앤 드롭 한 방에 결과 나오는 직관성
- **피해야 할 것**: AI 서비스 클리셰(위), 관공서식 딱딱함, 요소 빽빽하게 채우기

## 2. 색상

| 용도 | 값 | 비고 |
|---|---|---|
| 메인 (버튼, 강조) | 브랜드 그린 `#00C471` | 강조색은 이 초록 하나뿐 |
| 메인 hover | `#009E5F` | |
| 배경 | 흰색 `#FFFFFF` / 면 구분용 연회색 `#F2F4F6` | 토스식 — 그림자 대신 회색 면으로 구획 |
| 본문 텍스트 | 진남색 `#191F28` / 보조 `#8B95A1` | 토스 그레이스케일. 순검정(#000) 안 씀 |
| 초록 연한 배경 | `#E6F8EF` | 선택된 카드/칩의 배경 (초록 글씨와 세트) |
| 성공/에러 | 초록 `#16A34A` / 빨강 `#F04452` | 업로드 성공/실패 표시용 (성공 표시는 브랜드 그린과 겹치므로 체크 아이콘 위주로) |

- **강조색은 브랜드 그린 하나만**, 나머지는 전부 무채색 — 보라·그라데이션 금지 (AI 클리셰)
- 기존 예제 컴포넌트(TailwindExample)의 보라 버튼도 화면 제작 시 파랑으로 교체

## 3. 타이포그래피

- **폰트: Pretendard** (한글) — 웹폰트 링크 한 줄로 적용, 한/영 혼용에 강함
- 에디터 캔버스 안에서 쓰는 오버레이 폰트는 별개 (Google Fonts에서 영어권 힙한 폰트 / 일본어 귀여운 고딕 등 — 기능 요구사항)
- 크기 체계: 페이지 제목 32~36px / 섹션 제목 20~24px / 본문 16px / 보조 14px

## 4. 공통 컴포넌트 규칙

- 모서리: 카드 `rounded-2xl`~`rounded-3xl`, 버튼/입력 `rounded-xl` — 토스처럼 넉넉하게 둥글게
- 그림자: 기본적으로 안 씀 — **회색 면(`#F2F4F6`) 대비로 구획하는 게 토스식**. 모달 등 떠 있는 요소만 부드러운 그림자
- 버튼: 주요 액션은 초록 배경 + 흰 글씨, 보조 액션은 연회색 배경(`#F2F4F6`) + 진한 글씨 (테두리 버튼보다 토스식)
- **말투(마이크로카피)**: 전부 친근한 해요체 — "이모티콘을 올려주세요", "번역할 언어를 골라주세요". 기술 용어(OCR, Bounding Box) 화면에 노출 금지
- 아이콘: **lucide-react** 라이브러리로 통일 (개별 아이콘 파일 수집 금지)
- 로딩: AI 처리 중에는 반드시 **진행 상태 표시** (스피너 + "한글을 찾고 있어요…" 같은 단계별 문구) — AI 응답이 수 초 걸리므로 빈 화면 금지

## 5. 화면별 명세

### 5-1. 랜딩(메인) 페이지 `/`
- **목적**: 심사위원/방문자가 3초 안에 "우리 이런 서비스예요!"를 이해하고 [서비스 시작하기]를 누르게
- 위에서부터:
  1. **헤더**: 로고(왼쪽) + [시작하기] 버튼(오른쪽) — 메뉴는 최소화
  2. **히어로**: 큰 제목(슬로건: "K-이모티콘, 클릭 한 번으로 글로벌 밈이 되다") + 한 줄 설명 + 큰 [서비스 시작하기] 버튼
  3. **Before/After 데모**: "열공" 이모티콘 → "Grinding" 이모티콘 나란히 보여주기 (이거 하나로 서비스 이해 끝)
  4. **핵심 기능 2개 카드**: ① AI 초월 번역 ② 스마트 캔버스 (Plan.md의 Core Features)
  5. **푸터**: 팀명, GitHub 링크
- 분위기: 첫인상 담당. 여백 크게, Before/After 데모 이미지가 주인공
- **로그인 없음(no-login)**: [서비스 시작하기] / [시작하기] 버튼 → **바로 대시보드(5-2)로 이동.** 가입/로그인 문턱 없이 즉시 체험 — 심사위원 이탈 방지가 최우선

### 5-2. 대시보드(업로드) `/dashboard`
- **목적**: 이미지를 부담 없이 던져 넣게
- 구성:
  1. 화면 중앙에 **큰 드래그 앤 드롭 존** (점선 테두리, "PNG/GIF를 끌어다 놓으세요" + 클릭 업로드도 가능)
  2. 드롭 존은 **드래그 중 상태 변화 필수** (테두리·배경이 초록 톤으로 변함)
  3. 업로드된 이미지들은 아래에 **썸네일 그리드**로 프리뷰 (여러 장 동시 업로드 지원 — 가점 요소)
  4. **타겟 언어 선택**: 국기+언어명 카드 (🇺🇸 English / 🇯🇵 日本語 …) → [번역 시작] 버튼
- 상태 디자인 필요: 비어있음 / 드래그 중 / 업로드됨 / 처리 중(진행률)

### 5-3. 에디터 (핵심 화면) `/editor`
- **목적**: 이 서비스의 하이라이트. 시연의 90%가 이 화면에서 이뤄짐
- 레이아웃 (Plan.md UX Flow 그대로):
  - **왼쪽 패널**: 원본 이모티콘 (한글 텍스트에 감지된 Bounding Box 표시)
  - **오른쪽 패널**: 변환 미리보기 캔버스 (한글 지워지고 번역 텍스트 올라간 상태)
  - **하단 컨트롤러 바**:
    - 번역 후보 3개 칩(chip) — 클릭해서 선택, 선택된 것은 연초록 배경 + 초록 글씨
    - 폰트 선택 드롭다운 + 크기/회전 슬라이더 + 글자색 피커
  - 우상단: [다운로드] 버튼 (단일 PNG / 전체 ZIP 선택)
- **투명 배경 표시**: 캔버스 배경은 체커보드 패턴(포토샵처럼) — "투명하게 지워졌다"가 눈에 보여야 기술이 어필됨
- 캔버스 위 텍스트는 드래그로 이동/크기조절 가능 — 선택 시 조절 핸들 표시

### 5-4. (여유 있으면) 작업 내역
- 이전 작업 이어보기 — 카드 리스트. **우선순위 낮음, 마지막에**
- 로그인이 없으므로 "내 계정" 개념 없음 → 브라우저 저장(localStorage) 또는 익명 세션 ID 기반으로 같은 브라우저에서 재접속 시 이어보기 (Supabase 저장과 연결은 구현 시 결정)

## 6. 유의사항 (디자인이 지켜야 할 조건)

- **반응형**: 심사위원이 폰으로 열 수 있으니 **랜딩 페이지는 모바일 완벽 대응**. 대시보드는 안 깨지는 정도. **에디터는 데스크톱 전용**이어도 됨 (캔버스 드래그 조작은 폰에서 무리 — "PC에서 이용해주세요" 안내만 예쁘게)
- **Figma 프레임 기준: 1440px** — 콘텐츠는 가운데 최대 1200px 컨테이너 안에
- **다국어 길이 대응**: "열공"(2자) → "Grinding"(8자)처럼 번역 후 글자 수가 크게 변함. 텍스트 영역은 항상 여유 있게
- **밝은 테마 고정**: 발표는 프로젝터로 진행될 가능성 높음 → 어두운 화면은 프로젝터에서 잘 안 보임. 다크모드는 만들지 않음 (시간 절약)
- **AI 대기 시간**: OCR+번역에 수 초 소요 → 모든 AI 호출 구간에 로딩 UI 필수 (4번 참고)
- **데모 데이터**: 시연 때 쓸 샘플 이모티콘("열공", "대박", "인정")을 미리 준비해서 랜딩 Before/After와 시연에 재사용
- **무거운 리소스 금지**: 학교 와이파이 고려, 이미지 최적화 + 라이브러리 최소한만
- **no-login 원칙**: 가입/로그인 화면을 만들지 않음. 누구나 접속 즉시 업로드→변환→다운로드까지 가능해야 함. (남용 방지가 필요해지면 업로드 개수 제한 같은 가벼운 장치로 해결 — 로그인으로 해결하지 않기)

## 7. 작업 순서 제안 (디자인 관점)

1. 에디터 화면부터 (제일 어렵고 제일 중요 — 여기가 완성되면 나머지는 금방)
2. 대시보드 (업로드 UX)
3. 랜딩 페이지 (내용이 확정된 마지막에 만드는 게 두 번 일 안 함)
