# 🌐 글로벌 이모티콘 로컬라이저: Glocalizer 개발 계획서

> **대상:** 제4회 NAVER OGQ마켓 AI Competition 출품작
**슬로건:** "K-이모티콘, 클릭 한 번으로 글로벌 마켓의 인기 밈(Meme)이 되다."
> 

## 1. 서비스 컨셉 및 핵심 가치 (Value Proposition)

국내 창작자들이 만든 이모티콘을 해외 시장(영어권, 일본 등)에 출시할 때 가장 큰 장벽은 '문화적 번역(Localization)'과 '이미지 수정 작업'입니다.
`Glocalizer`는 이 두 가지 귀찮고 어려운 과정을 AI를 통해 단 몇 초 만에 해결하여, 창작자가 글로벌 마켓(OGQ 글로벌, 라인 등)으로 즉시 유통망을 확장할 수 있도록 돕는 웹 기반 SaaS 프로덕트입니다.

## 2. 핵심 기능 2가지 (Core Features)

가장 직관적이고 구현 가능한 핵심 기능 두 가지에 역량을 집중합니다.

### 1) AI 문화적 뉘앙스 번역 및 텍스트 감지 (OCR & Cultural LLM)

단순한 번역기(Papago, Google Translate)는 한국어 신조어나 밈의 맛을 살리지 못합니다. AI를 활용해 이미지 속 한국어를 감지하고, 해당 국가의 Z세대가 쓸 법한 현지 밈(Meme)으로 초월 번역합니다.

- **동작 방식:**
    1. 사용자가 한국어 텍스트가 포함된 이모티콘 이미지(PNG/GIF)를 업로드합니다.
    2. **OCR API**를 통해 이미지 내 한글의 위치(Bounding Box)와 텍스트 내용을 추출합니다.
    3. 추출된 텍스트를 LLM(Claude 3.5 Sonnet 또는 GPT-4o)에 전달하여 타겟 국가(영어/일본어 등)의 감성과 뉘앙스에 맞는 후보군 3개를 생성합니다.
- **초월 번역 예시:**
    - *"열공" -> "Grinding" (단순 번역 "Hard study" 대신 Z세대 표현 사용)*
    - *"대박" -> "No way!" 또는 "Slay!"*
    - *"인정" -> "Fr" (For Real)*

### 2) 스마트 캔버스 & 폰트 스타일러 (Smart Eraser & Font Overlay)

이미지에서 기존 한글을 깔끔하게 지우고, 그 자리에 번역된 다국어 텍스트를 자연스럽게 얹어주는 기능입니다.

- **현실적이고 똑똑한 기술적 우회 (Smart Shortcut):** /
    - *어려운 길:* Stable Diffusion Inpainting을 서버에 구축하기 (2주 안에 개발 및 배포하기에는 서버 비용과 인프라 설정이 너무 무거움).
    - *쉽고 확실한 길:* 이모티콘은 보통 배경이 투명(Alpha Channel)이거나 단색입니다.
        1. OCR로 검출된 텍스트 영역(Bounding Box)의 픽셀을 HTML Canvas를 이용해 투명하게 지우거나 배경색으로 채웁니다. (Frontend에서 100% 처리 가능)
        2. 그 위에 Google Fonts API를 연동하여 번역된 다국어 텍스트를 올려줍니다.
        3. 사용자가 웹 브라우저 화면상에서 텍스트의 크기, 회전, 폰트 종류, 글자 색상을 마우스 드래그로 미세 조정(Tweak)할 수 있도록 합니다.

## 3. 유저 시나리오 (UX Flow)

1. **대시보드:** 이모티콘 이미지(PNG)를 드래그 앤 드롭으로 업로드합니다. (여러 장 동시 업로드 가능하게 설계하면 가점)
2. **타겟 언어 선택:** 번역하고자 하는 언어(영어, 일본어 등)를 선택합니다.
3. **에디터 화면 (핵심):**
    - **왼쪽:** 원본 이모티콘 이미지
    - **오른쪽:** 번역 및 글씨가 지워진 미리보기 이미지
    - **하단 컨트롤러:** 번역 추천 단어 선택(3개 중 택 1), 글꼴(Font) 스타일 변경 슬라이더.
4. **다운로드:** 완성된 글로벌 버전 이모티콘을 원본 파일명 뒤에 `_en` 등이 붙은 형태로 단일 다운로드 또는 ZIP 압축 다운로드합니다.

# 🌐 Glocalizer 기술 스택 매핑 및 상세 설계서

### 1. 🎨 디자인 및 UI/UX 개발 (Figma / Claude Design)

- **Figma:** 대시보드(Drag & Drop), 에디터 화면(좌: 원본, 우: 캔버스, 하단: 컨트롤러)의 와이어프레임과 컴포넌트를 설계해. Tailwind CSS의 그리드 시스템(`grid-cols-12`)을 염두에 두고 레이아웃을 잡으면 좋아.
- **Claude Design (Artifacts):** Figma로 프로토타입을 짜기 전, Claude에게 "Tailwind CSS와 Lucide React 아이콘을 사용한 대시보드 및 이모티콘 에디터 UI 코드 짜줘"라고 요청해서 컴포넌트 구조의 영감을 빠르게 얻고 코드로 직행할 수 있어.

### 2. 💻 프론트엔드 아키텍처 (React + Vite + TS + Tailwind CSS)

사용자에게 보여지는 핵심 에디터와 스티커 처리 엔진을 담당해.

- **대시보드 (파일 업로드):**
    - `react-dropzone` 라이브러리를 활용해 여러 장의 PNG/GIF 이미지를 동시에 드래그 앤 드롭으로 업로드받아.
    - 업로드된 이미지들은 `URL.createObjectURL(file)`을 통해 화면에 프리뷰로 먼저 보여주고, 백엔드로 전송해.
- **스마트 캔버스 에디터 (핵심 기능 2):**
    - 오른쪽의 미리보기 화면은 HTML5 `<canvas>` 태그를 사용하거나, React와 궁합이 좋은 `fabric.js` 혹은 `konva` 라이브러리를 활용해.
    - **글자 지우기 알고리즘:** 백엔드 AI가 보내준 한글의 `bounding_box` 좌표(`x, y, width, height`)를 받아, Canvas API의 `ctx.clearRect(x, y, width, height)`를 실행해. 투명 배경(Alpha 채널)을 유지하면서 글자만 마법처럼 투명하게 날려버릴 수 있어.
    - **텍스트 오버레이:** 지워진 영역 위에 `Google Fonts API`로 연동된 현지 폰트(예: 영어는 힙한 압축 폰트, 일본어는 귀여운 고딕체)를 올리고, 사용자가 드래그로 크기/회전(`transform`)을 조절할 수 있게 바인딩해.
- **글로벌 다운로드 엔진:**
    - 수정이 완료된 Canvas 객체를 `canvas.toDataURL('image/png')`를 통해 이미지 데이터로 추출해.
    - `jszip` 라이브러리를 사용해 프론트엔드단에서 즉시 원본파일명 뒤에 `_en`, `_ja`를 붙여 ZIP 파일로 압축 후 유저에게 다운로드해 줘. (서버 리소스를 전혀 쓰지 않는 쾌적한 UX 구현 가능)

### 3. ⚙️ 백엔드 및 데이터베이스 (Node.js v22 + Express + TS + Supabase)

안정적인 데이터 처리와 AI 오케스트레이션을 담당해.

- **런타임 환경:** 최신 **Node.js v22** 환경을 활용해 `fetch` API 등을 기본적으로 사용하며 비동기 처리(`async/await`) 속도를 최적화해.
- **Supabase Storage:** 유저가 업로드한 원본 이모티콘 이미지와 최종 변환된 결과물 이미지를 보관하는 버킷(`emoticons`)을 관리해.
- **Supabase DB:** 프로젝트 정보(`projects`)와 이모티콘 상태 및 AI가 분석한 좌표 데이터(`emoticons` 테이블의 `ocr_translation_data` 컬럼에 JSONB 형태로 저장)를 관리해. 추후 유저가 재접속해도 작업 내역을 이어볼 수 있게 해주는 차별점이 돼.
- **Express Router:**
    - `POST /api/translate`: 이미지를 받아 Supabase Storage에 저장하고, AI 엔진을 호출해 좌표 및 번역 결과를 받아 DB에 기록한 뒤 프론트엔드로 리턴하는 핵심 API 라우트.

### 4. 🧠 서비스 AI 파이프라인 (Gemini + ChatGPT + Claude)

하나의 모델만 쓰는 것보다 **"Multi-AI 오케스트레이션"** 구조를 가져가면 기술 점수에서 엄청난 가산점을 받아.

```
[원본 이미지]
   │
   ├──► 1단계: Gemini 2.5 Flash / GPT-4o (Multimodal)
   │     - 이미지 속 한글의 정확한 픽셀 좌표(Bounding Box) 검출
   │     - 단순 텍스트 추출 (OCR)
   │
   └──► 2단계: Claude 3.5 Sonnet (Cultural Translation)
         - 추출된 한글 텍스트와 컨셉을 바탕으로, Target 국가의 Z세대가 쓸 법한
           "가장 트렌디한 현지 밈(Meme) 후보군 3개" 및 스타일에 대한 설명 생성
```

- **Gemini/GPT-4o (비전 담당):** 이미지 분석 비용이 가장 저렴하고 속도가 빠른 **Gemini 2.5 Flash**나 **GPT-4o**에 이미지 바이너리를 보내 텍스트와 좌표를 추출해.
- **Claude 3.5 Sonnet (뉘앙스 번역 담당):** "열공", "대박", "인정" 같은 단어를 Claude에게 전달하며 *"너는 미국 현지 웹툰 번역 전문가야. 이 단어들을 현재 미국 Z세대들이 틱톡이나 디스코드에서 쓰는 가장 힙한 밈(Slang) 3가지로 맛깔나게 초월 번역해 줘"*라고 프롬프팅을 수행해.
- 최종적으로 백엔드는 프론트엔드가 바로 Canvas에 그릴 수 있도록 좌표와 Claude의 번역 후보군 3개가 합쳐진 **정형화된 JSON 데이터**를 리턴해 주는 거지.

### 5. 🛠️ 생산성 및 배포 환경 (VS Code + Claude Code CLI + GitHub + Vercel)

- **Claude Code CLI:** 터미널 창에서 Claude를 바로 호출하는 도구야. 예를 들어, 프론트엔드 개발 중에 `Claude Code`에게 *"이 Canvas 좌표 기반으로 fabric.js 텍스트박스 중앙 정렬하는 헬퍼 함수 작성해 줘"*라고 하면 코드를 터미널에서 즉시 짜주고 파일에 적용해 주니 개발 속도가 3배는 빨라질 거야.
- **GitHub & Vercel 배포:**
    - 레포지토리를 백엔드와 프론트엔드가 합쳐진 모노레포(Monorepo) 혹은 독립 구조로 세팅해.
    - Vercel에 연동하면 GitHub `main` 브랜치에 `push`할 때마다 자동으로 빌드 및 배포가 이뤄져서, 상시 접속 가능한 **라이브 프로덕트 URL**을 심사위원에게 즉시 제공할 수 있어.
